# Security Policy

## Reporting a Vulnerability

If you believe you have found a security vulnerability in veez-pii-guard, do not open a public issue.

Please report it privately by email to security@veez.ai with:

- A short description of the issue
- Steps to reproduce
- Potential impact
- Any proof-of-concept or logs that help confirm the finding

We will acknowledge receipt as quickly as possible, investigate the report, and coordinate disclosure once a fix is available.

## Scope

This repository is focused on local PII detection and anonymization. Reports are especially helpful for:

- False negatives that expose sensitive data
- False positives that break expected usage
- Denial-of-service or crash vectors
- Unsafe defaults in the CLI or library API
- Supply-chain or dependency concerns

## Disclosure

Please give maintainers reasonable time to investigate and ship a fix before public disclosure.
